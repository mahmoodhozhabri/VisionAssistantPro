# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый мультимодальный AI-ассистент для NVDA. Он использует передовые AI-движки для интеллектуального чтения экрана, перевода, голосового ввода и анализа документов.

_Это дополнение было выпущено для сообщества в честь Международного дня инвалидов._

## 1. Установка и настройка

Перейдите в **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый AI-сервис. Поддерживаются **Google Gemini**, **OpenAI**, **Mistral**, **Groq** и **Custom** (совместимые с OpenAI серверы, такие как Ollama/LM Studio).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно при анализе изображений/файлов).
- **API Key:** Обязательное поле. Вы можете ввести несколько ключей (разделяя их запятыми или новыми строками) для автоматической ротации.
- **Fetch Models:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить актуальный список доступных моделей от провайдера.
- **AI Model:** Выберите основную модель для общего чата и анализа.

### 1.2 Продвинутая маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Custom.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для продвинутых пользователей**. Если вы не уверены, что делает конкретная модель, оставьте этот пункт **неотмеченным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) вызовет ошибки и остановит работу дополнения.

Отметьте **"Advanced Model Routing (Task-specific)"**, чтобы получить детальный контроль. Это позволит вам выбирать конкретные модели из выпадающего списка для разных задач:
- **OCR / Vision Model:** Выберите специализированную модель для анализа изображений.
- **Speech-to-Text (STT):** Выберите конкретную модель для диктовки.
- **Text-to-Speech (TTS):** Выберите модель для генерации аудио.
- **AI Operator Model:** Выберите конкретную модель для задач автономного управления компьютером.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Продвинутая настройка конечных точек (Custom Provider)
*Доступно только при выборе "Custom".*

> **⚠️ Предупреждение:** Этот раздел предназначен для ручной настройки API и рассчитан на **опытных пользователей**, запускающих локальные серверы или прокси. Неправильные URL или имена моделей нарушат соединение. Если вы точно не знаете, что делают эти конечные точки, оставьте этот пункт **неотмеченным**.

Отметьте **"Advanced Endpoint Configuration"**, чтобы вручную ввести данные сервера. В отличие от родных провайдеров, здесь необходимо **вводить** конкретные URL и имена моделей:
- **Models List URL:** URL для получения доступных моделей.
- **OCR/STT/TTS Endpoint URL:** Полные URL для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Custom Models:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.4 Общие настройки
- **OCR Engine:** Выберите между **Chrome (Fast)** для быстрых результатов или **AI (Advanced)** для лучшего сохранения форматирования.
    - *Примечание:* Если вы выберете "AI (Advanced)", но ваш провайдер установлен на OpenAI/Groq, дополнение интеллектуально направит изображение в визуальную модель вашего активного провайдера.
- **TTS Voice:** Выберите предпочитаемый стиль голоса. Этот список обновляется динамически в зависимости от вашего активного провайдера.
- **Creativity (Temperature):** Управляет случайностью ответов AI. Более низкие значения лучше подходят для точного перевода/OCR.
- **Proxy URL:** Настройте это, если AI-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, такие как `127.0.0.1` или bridge URL).

## 2. Командный слой и горячие клавиши

Чтобы избежать конфликтов клавиатуры, это дополнение использует **Командный слой**.
1. Нажмите **NVDA + Shift + V** (основная клавиша) для активации слоя (вы услышите звуковой сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих клавиш:

| Клавиша       | Функция                  | Описание                                                                                             |
|---------------|--------------------------|------------------------------------------------------------------------------------------------------|
| **Shift + A** | **AI Operator**          | **Автономное управление:** Дайте AI инструкцию выполнить задачу на вашем экране.                      |
| **E**         | **UI Explorer**          | **Интерактивный клик:** Определяет и нажимает элементы UI в любом приложении.                         |
| **T**         | Smart Translator         | Переводит текст под курсором навигатора или выделенный текст.                                         |
| **Shift + T** | Clipboard Translator     | Переводит содержимое буфера обмена.                                                                  |
| **R**         | Text Refiner             | Суммирует, исправляет грамматику, объясняет или выполняет **Пользовательские промпты**.               |
| **V**         | Object Vision            | Описывает текущий объект навигатора.                                                                 |
| **O**         | Full Screen Vision       | Анализирует весь макет и содержимое экрана.                                                          |
| **Shift + V** | Online Video Analysis    | Анализ видео с **YouTube**, **Instagram**, **TikTok** или **Twitter (X)**.                            |
| **D**         | Document Reader          | Продвинутое чтение PDF и изображений с выбором диапазона страниц.                                   |
| **F**         | **Smart File Action**    | Контекстно-зависимое распознавание выбранных файлов изображений, PDF или TIFF.                       |
| **A**         | Audio Transcription      | Преобразует MP3, WAV или OGG файлы в текст.                                                         |
| **C**         | CAPTCHA Solver           | Захватывает и решает CAPTCHA (поддерживает правительственные порталы).                               |
| **S**         | Smart Dictation          | Преобразует речь в текст. Нажмите для начала записи, снова нажмите для остановки/ввода.             |
| **I**         | Status Reporting         | Сообщает текущий прогресс (например, "Сканирование...", "Ожидание").                                 |
| **L**         | **Label Object**         | **Семантическая AI-маркировка:** Постоянно маркирует текущий сфокусированный элемент/иконку.        |
| **Shift + L** | **Manage/Scan Labels**   | Открывает Менеджер меток (если метки существуют) или сканирует приложение на наличие безымянных элементов. |
| **U**         | Update Check             | Проверяет вручную на GitHub наличие последней версии дополнения.                                     |
| **Space**     | Recall Last Result       | Показывает последний ответ AI в диалоговом окне для просмотра или уточнения.                         |
| **H**         | Commands Help            | Отображает список всех доступных горячих клавиш.                                                     |

### 2.1 Горячие клавиши Document Reader (внутри окна просмотра)
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть диалог чата, чтобы задать вопросы о документе.
- **Alt + R:** Принудительно **повторно сканировать с помощью AI**, используя вашего активного провайдера.
- **Alt + G:** Сгенерировать и сохранить высококачественный аудиофайл (WAV/MP3). *Скрыто, если провайдер не поддерживает TTS.*
- **Alt + S / Ctrl + S:** Сохранить извлеченный текст как TXT или HTML файл.

## 3. Пользовательские промпты и переменные

Вы можете управлять промптами в **Настройки > Промпты > Управление промптами...**.

### Поддерживаемые переменные
- `[selection]`: Текущий выделенный текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Скриншот объекта навигатора.
- `[screen_full]`: Полный скриншот экрана.
- `[file_ocr]`: Выбрать файл изображения или PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, код, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех AI-функций требуется активное подключение к Интернету. Многостраничные документы обрабатываются автоматически.

## Изменения в версии 6.0

*   **Представлена семантическая AI-маркировка**: Пользователи теперь могут постоянно маркировать безымянные кнопки и значки с помощью AI. Нажмите **L**, чтобы пометить текущий объект навигатора (поддерживается как фокус вкладок, так и навигация по объектам), или **Shift+L**, чтобы отсканировать и пометить сразу все приложение.
*   **Интеллектуальное управление метками**: Добавлен новый, полностью доступный диалог Менеджера меток (через **Shift+L**, если метки существуют) для просмотра, переименования или массового удаления пользовательских меток.
*   **Прямой анализ файлов (в обход диалога выбора файла)**: Дополнение теперь достаточно умно, чтобы определить, фокусируетесь ли вы на файле PDF или изображения в проводнике Windows. Нажатие **F (Smart File Action)** или **D (Document Reader)** на выделенном файле немедленно обработает его, полностью минуя стандартный диалог "Открыть".

## Изменения в версии 5.6

*   **Добавлен движок OCR "None (Extract Text Layer)"**: Пользователи теперь могут извлекать текст напрямую из PDF с текстовым слоем без использования AI-кредитов, что значительно повышает скорость и конфиденциальность при работе с текстовыми документами.
*   **Улучшена точность UI Explorer**: Улучшен промпт UI Explorer для лучшего определения типов элементов (например, элементов списка) и точного сообщения состояний, таких как "(Отмечено)", "(Выбрано)" или "(Развернуто)", при этом игнорируются системные компоненты Windows, такие как Панель задач и Часы.
*   **Напоминание при установке**: Добавлено уведомление после установки, которое направляет пользователей в меню настроек для ввода их API-ключей и предпочтений.

## Изменения в версии 5.5.2

*   **Исправлена проблема с вводом AI Operator**: Устранена ошибка, из-за которой на некоторых системах вместо вставки текста печаталась буква 'v'. Это исправление устраняет проблемы синхронизации, возникающие при высокой нагрузке на систему.
*   **Повышенная стабильность**: Добавлена надежная обработка ошибок для операций с буфером обмена, чтобы предотвратить сбои дополнения, когда системный буфер обмена временно заблокирован другими приложениями.
*   **Оптимизация синхронизации**: Скорректированы внутренние задержки для событий клавиатуры, чтобы обеспечить более высокую надежность на разных скоростях работы системы и лучшую совместимость со сторонними менеджерами буфера обмена.

## Изменения в версии 5.5 (Обновление автоматизации)

*   **AI Operator (Автономное управление - Shift+A)**: Это главная жемчужина версии 5.5. Vision Assistant Pro перешел от пассивного помощника к вашему личному **AI-оператору**. Он не просто описывает экран — он берет управление на себя.
    *   *Как это работает:* Теперь вы можете давать голосовые инструкции для управления вашим ПК. Например, в полностью недоступном приложении, где ваш скринридер молчит, вы можете нажать **Shift+A** и ввести: *"Нажми на кнопку 'Настройки'"* или *"Найди поле поиска, введи 'Последние новости' и нажми enter"*. AI визуально идентифицирует элементы, перемещает мышь и выполняет задачу за вас.
    *   *Примечание по производительности:* Эта функция оптимизирована для **Gemini 3.0 Flash (Preview)**, обеспечивая невероятно быстрые и интеллектуальные ответы, способные обрабатывать даже самые сложные UI-макеты.
    *   **⚠️ Предупреждение об использовании API:** Поскольку AI-оператору нужно "видеть", что именно происходит, для точности, он отправляет скриншот высокого разрешения с каждым шагом. Учтите, что частое использование будет потреблять вашу API-квоту намного быстрее, чем стандартные текстовые задачи.
*   **Визуальный UI Explorer (E)**: Устали от навигации по "немаркированным кнопкам"? Нажмите **E**, чтобы активировать UI Explorer. AI просканирует все окно и создаст список всех кликабельных элементов, которые он видит — включая значки, графику и меню. Просто выберите элемент из списка, и AI-оператор нажмет на него за вас. Это как иметь "доступный слой" поверх любого приложения.
*   **Контекстно-зависимое Smart File Action (F)**: Клавиша "F" была полностью переработана. Она больше не предполагает, что вам нужен только OCR. Когда вы выбираете одно изображение, она теперь интеллектуально спрашивает о вашем намерении: вы можете выбрать **Детальное визуальное описание**, чтобы понять сцену, или **Структурированное извлечение текста (OCR)** для чтения. Меню динамически адаптируется в зависимости от типа файла и вашего активного AI-движка.
*   **Оптимизация ядра**: Мы провели глубокую очистку внутренней логики дополнения, удалив неиспользуемые устаревшие функции и избыточный код. Это обеспечивает более легкий, быстрый и надежный опыт для всех пользователей.
