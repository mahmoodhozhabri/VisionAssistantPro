# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый многофункциональный AI-ассистент для NVDA. Он использует лучшие в мире AI-движки для интеллектуального чтения с экрана, перевода, голосового диктовки и анализа документов.

_Это дополнение было выпущено для сообщества в честь Международного дня инвалидов._

## 1. Установка и настройка

Перейдите в **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый AI-сервис. Поддерживаемые провайдеры: **Google Gemini**, **OpenAI**, **Mistral**, **Groq** и **Custom** (серверы, совместимые с OpenAI, такие как Ollama, LM Studio, Jan.ai или KoboldCPP).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений и файлов).
- **API Key:** Обязателен. Вы можете ввести несколько ключей (разделяя их запятыми или новыми строками) для автоматической ротации.
- **Fetch Models:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить последний список доступных моделей от провайдера.
- **AI Model:** Выберите основную модель, используемую для общего чата и анализа.

### 1.2 Продвинутая маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Custom.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для продвинутых пользователей**. Если вы не уверены, что делает конкретная модель, оставьте этот параметр **выключенным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) вызовет ошибки и остановит работу дополнения.

Отметьте **«Advanced Model Routing (Task-specific)»**, чтобы получить детальный контроль. Это позволит вам выбирать конкретные модели из раскрывающегося списка для различных задач:
- **OCR / Vision Model:** Выберите специализированную модель для анализа изображений.
- **Speech-to-Text (STT):** Выберите конкретную модель для диктовки.
- **Text-to-Speech (TTS):** Выберите модель для генерации аудио.
- **AI Operator Model:** Выберите конкретную модель для задач автономного управления компьютером.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Расширенная конфигурация конечных точек (провайдер Custom)
*Доступно только при выборе «Custom».*

> **⚠️ Предупреждение:** Этот раздел позволяет вручную настраивать API и предназначен для **опытных пользователей**, запускающих локальные серверы или прокси. Неправильные URL-адреса или имена моделей нарушат соединение. Если вы точно не знаете, что делают эти конечные точки, оставьте этот параметр **выключенным**.

Отметьте **«Advanced Endpoint Configuration»**, чтобы вручную ввести данные сервера. В отличие от стандартных провайдеров, здесь вы должны **ввести** конкретные URL-адреса и имена моделей:
- **Models List URL:** Конечная точка для получения доступных моделей.
- **OCR/STT/TTS Endpoint URL:** Полные URL-адреса для конкретных служб (например, `http://localhost:11434/v1/audio/speech`).
- **Custom Models:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.3.1 Настройка локального AI (однодейственная конфигурация)
Чтобы сделать интеграцию локального полностью офлайн AI максимально простой, внутри настроек Custom Provider доступна специальная кнопка **«Setup Local AI»**.

Если на вашем компьютере запущен сервер локальной AI-модели:
1. Выберите **Custom** в качестве провайдера.
2. Нажмите кнопку **Setup Local AI**.
3. Выберите ваш локальный AI-движок из доступного диалога:
   - **Ollama** (по умолчанию `http://127.0.0.1:11434`)
   - **LM Studio** (по умолчанию `http://127.0.0.1:1234`)
   - **Jan.ai** (по умолчанию `http://127.0.0.1:1337`)
   - **KoboldCPP** (по умолчанию `http://127.0.0.1:5001`)
4. Дополнение мгновенно настроит правильный локальный URL, тип API и автоматически получит ваши активные офлайн-модели для заполнения поля выбора **AI Model**.

*Примечание о сети и прокси: Этот механизм локального подключения имеет расширенный механизм обхода прокси. Даже если у вас активен системный VPN или TUN-режим прокси, запросы к вашему локальному AI будут полностью обходить его, обеспечивая стабильные офлайн-подключения без ошибок 502 Bad Gateway.*

### 1.4 Общие настройки
- **OCR Engine:** Выберите между **Chrome (Fast)** для быстрых результатов или **AI (Advanced)** для превосходного сохранения макета.
    - *Примечание:* Если вы выберете «AI (Advanced)», но ваш провайдер установлен на OpenAI/Groq, дополнение интеллектуально направит изображение на визуальную модель вашего активного провайдера.
- **TTS Voice:** Выберите предпочтительный стиль голоса. Этот список динамически обновляется в зависимости от вашего активного провайдера.
- **Creativity (Temperature):** Управляет случайностью AI. Более низкие значения лучше подходят для точного перевода/OCR.
- **Proxy URL:** Настройте это, если AI-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, такие как `127.0.0.1`, или мостовые URL-адреса).

## 2. Командный слой и горячие клавиши

Чтобы избежать конфликтов клавиш, это дополнение использует **Командный слой**.
1. Нажмите **NVDA + Shift + V** (основная клавиша), чтобы активировать слой (вы услышите звуковой сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих одиночных клавиш:

| Клавиша      | Функция                  | Описание                                                                                                 |
|--------------|--------------------------|----------------------------------------------------------------------------------------------------------|
| **Shift + A**| **AI Operator**          | **Автономное управление:** Сообщите AI задачу, которую нужно выполнить на вашем экране.                     |
| **E**        | **UI Explorer**          | **Интерактивный клик:** Определяет и кликает по элементам интерфейса в любом приложении.                    |
| **T**        | Smart Translator         | Переводит текст под курсором навигатора или выделенный текст.                                               |
| **Shift + T**| Clipboard Translator     | Переводит содержимое буфера обмена.                                                                       |
| **R**        | Text Refiner             | Суммирует, исправляет грамматику, объясняет или выполняет **Пользовательские промпты**.                    |
| **V**        | Object Vision            | Описывает текущий объект навигатора.                                                                      |
| **O**        | Full Screen Vision       | Анализирует весь макет и содержимое экрана.                                                              |
| **Shift + V**| Online Video Analysis    | Анализирует видео с **YouTube**, **Instagram**, **TikTok** или **Twitter (X)**.                            |
| **D**        | Document Reader          | Расширенное чтение PDF и изображений с выбором диапазона страниц.                                          |
| **F**        | **Smart File Action**    | Контекстно-зависимое распознавание выбранного файла изображения, PDF или TIFF.                              |
| **A**        | Audio Transcription      | Транскрибирует файлы MP3, WAV или OGG в текст.                                                             |
| **C**        | CAPTCHA Solver           | Захватывает и решает CAPTCHA (поддерживаются государственные порталы).                                      |
| **S**        | Smart Dictation          | Преобразует речь в текст. Нажмите, чтобы начать запись, нажмите снова, чтобы остановить/ввести текст.       |
| **I**        | Status Reporting         | Сообщает текущий прогресс (например, «Сканирование...», «Ожидание»).                                      |
| **L**        | **Label Object**         | **Семантическая AI-маркировка:** Постоянно маркирует текущий сфокусированный элемент/значок.               |
| **Shift + L**| **Manage/Scan Labels**   | Открывает Менеджер меток (если метки существуют) или сканирует приложение на наличие безымянных элементов.     |
| **U**        | Update Check             | Вручную проверяет GitHub на наличие последней версии дополнения.                                            |
| **Space**    | Recall Last Result       | Показывает последний ответ AI в диалоговом окне чата для просмотра или продолжения.                          |
| **H**        | Commands Help            | Отображает список всех доступных горячих клавиш.                                                           |

### 2.1 Горячие клавиши Document Reader (внутри просмотрщика)
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть диалог чата, чтобы задать вопросы о документе.
- **Alt + R:** Принудительно **повторно сканировать с помощью AI**, используя вашего активного провайдера.
- **Alt + G:** Сгенерировать и сохранить высококачественный аудиофайл (WAV/MP3). *Скрыто, если провайдер не поддерживает TTS.*
- **Alt + S / Ctrl + S:** Сохранить извлеченный текст в файл TXT или HTML.

## 3. Пользовательские промпты и переменные

Вы можете управлять промптами в **Настройки > Промпты > Управление промптами...**.

### Поддерживаемые переменные
- `[selection]`: Выделенный в данный момент текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Снимок экрана объекта навигатора.
- `[screen_full]`: Полноэкранный снимок экрана.
- `[file_ocr]`: Выбрать файл изображения/PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, Code, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех AI-функций требуется активное подключение к Интернету. Многостраничные документы обрабатываются автоматически.

## Изменения в версии 6.1.0

*   **Универсальная интеграция локального ИИ (Настройка локального ИИ)**: Добавлена новая кнопка **«Setup Local AI»** в настройках пользовательского провайдера. Теперь пользователи могут автоматически настраивать локальные ИИ-движки, включая **Ollama**, **LM Studio**, **Jan.ai** и **KoboldCPP**, мгновенно.
*   **Интеллектуальный обход локального прокси**: Переработана логика подключения с расширенным механизмом обхода прокси. Теперь дополнение достаточно умно, чтобы полностью обходить системные прокси Windows для локальных зацикленных соединений, обеспечивая стабильное подключение к локальному ИИ даже при активном VPN/TUN-режиме.
*   **Аварийная остановка ИИ-Оператора (Shift+A)**: Добавлен долгожданный предохранитель остановки/отмены. Нажатие команды ИИ-Оператора (**Shift+A** внутри командного слоя**) во время выполнения автономной операции мгновенно прервет цикл и сообщит *«ИИ-Оператор остановлен»*.
*   **Сверхстабильная ИИ-маркировка (v2)**: Заменил абсолютные координаты экрана на передовую гибридную систему **Object Signature** (сигнатуры объекта). Теперь метки полагаются на программные идентификаторы (UIA **AutomationId** или Win32 **ControlID**) и координаты относительно окна, что делает ваши пользовательские метки полностью устойчивыми к изменению размера окна, перемещению, переключению мониторов или масштабированию.
*   **Бесшовная автоматическая миграция меток**: Обновление происходит полностью прозрачно. Дополнение автоматически перенесет ваши старые метки, основанные на координатах, в новый стабильный формат «отпечатков» в фоновом режиме при первом фокусе, без потери данных.

## Изменения в версии 6.0

*   **Представлена семантическая AI-маркировка**: Теперь пользователи могут постоянно маркировать безымянные кнопки и значки с помощью AI. Нажмите **L**, чтобы маркировать текущий объект навигатора (поддерживается как фокус вкладок, так и навигация по объектам), или **Shift+L**, чтобы сканировать и маркировать все приложение сразу.
*   **Интеллектуальное управление метками**: Добавлен новый, полностью доступный диалог «Менеджер меток» (доступен по **Shift+L**, если метки существуют) для просмотра, переименования или массового удаления пользовательских меток.
*   **Прямой анализ файлов (обход диалога выбора файла)**: Теперь дополнение достаточно умно, чтобы определить, фокусируетесь ли вы в данный момент на файле PDF или изображения в проводнике Windows. Нажатие **F (Smart File Action)** или **D (Document Reader)** на выделенном файле немедленно обработает его, полностью минуя стандартный диалог «Открыть».

*   **Напоминание при установке**: Добавлено уведомление после установки, которое направляет пользователей в меню настроек для настройки их API-ключей и предпочтений.

## Изменения в версии 5.5.2

*   **Исправлена проблема с вводом AI Operator**: Устранена ошибка, из-за которой на некоторых системах вводилась буква «v» вместо вставки текста. Это исправление устраняет конфликты тайминга, возникающие при высокой нагрузке на систему.
*   **Повышенная стабильность**: Добавлена надежная обработка ошибок для операций с буфером обмена, чтобы предотвратить сбои дополнения, когда системный буфер обмена временно заблокирован другими приложениями.
*   **Оптимизация тайминга**: Скорректированы внутренние задержки для событий клавиатуры, чтобы обеспечить более высокую надежность на разных скоростях системы и лучшую совместимость со сторонними менеджерами буфера обмена.

## Изменения в версии 5.5 (Обновление автоматизации)

*   **AI Operator (Автономное управление - Shift+A)**: Это жемчужина версии 5.5. Vision Assistant Pro перешел от пассивного помощника к вашему личному **AI-оператору**. Он не просто описывает экран — он берет управление на себя.
    *   *Как это работает:* Теперь вы можете давать голосовые инструкции для управления вашим ПК. Например, в совершенно недоступном приложении, где ваша программа чтения с экрана молчит, вы можете нажать **Shift+A** и ввести: *«Нажми на кнопку "Настройки"»* или *«Найди поле поиска, введи "Последние новости" и нажми Enter»*. AI визуально идентифицирует элементы, перемещает мышь и выполняет задачу за вас.
    *   *Примечание о производительности:* Эта функция оптимизирована для **Gemini 3.0 Flash (Preview)**, обеспечивая невероятно быстрые и интеллектуальные ответы, которые могут справиться даже с самыми сложными макетами интерфейса.
    *   **⚠️ Предупреждение об использовании API:** Поскольку AI-оператору нужно «видеть», что происходит, чтобы быть точным, он отправляет полноразмерный снимок экрана с каждым шагом. Учтите, что частое использование будет потреблять вашу квоту API намного быстрее, чем стандартные текстовые задачи.
*   **Визуальный UI Explorer (E)**: Устали от навигации по «немаркированным кнопкам»? Нажмите **E**, чтобы активировать UI Explorer. AI просканирует все окно и сгенерирует список каждого кликабельного элемента, который он видит, включая значки, графику и меню. Просто выберите элемент из списка, и AI-оператор нажмет на него за вас. Это как иметь «доступный слой» поверх любого приложения.
*   **Контекстно-зависимое Smart File Action (F)**: Клавиша «F» была полностью переработана. Она больше не предполагает, что вам нужен только OCR. Когда вы выбираете одно изображение, она теперь интеллектуально спрашивает о вашем намерении: вы можете выбрать **Детальное визуальное описание**, чтобы понять сцену, или **Структурированное извлечение текста (OCR)** для чтения. Меню динамически адаптируется в зависимости от типа файла и вашего активного AI-движка.
*   **Оптимизация ядра**: Мы провели глубокую очистку внутренней логики дополнения, удалив неиспользуемые устаревшие функции и избыточный код. Это приводит к более легкому, быстрому и надежному опыту для всех пользователей.
