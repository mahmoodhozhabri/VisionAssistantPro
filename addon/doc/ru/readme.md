# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый многофункциональный AI-ассистент для NVDA. Он использует ведущие мировые AI-движки для обеспечения интеллектуального чтения с экрана, перевода, голосового ввода и анализа документов.

_Это дополнение было выпущено для сообщества в честь Международного дня людей с инвалидностью._

## 1. Настройка и конфигурация

Перейдите в **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый AI-сервис. Поддерживаемые провайдеры включают **Google Gemini**, **OpenAI**, **Mistral**, **Groq** и **Custom** (OpenAI-совместимые серверы, такие как Ollama/LM Studio).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений/файлов).
- **API-ключ:** Обязателен для заполнения. Вы можете ввести несколько ключей (разделяя их запятыми или новыми строками) для автоматической ротации.
- **Загрузить модели:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить актуальный список доступных моделей от провайдера.
- **AI-модель:** Выберите основную модель, используемую для общего чата и анализа.

### 1.2 Продвинутая маршрутизация моделей (Нативные провайдеры)
*Доступно для Gemini, OpenAI, Groq и Mistral.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для продвинутых пользователей**. Если вы не уверены, для чего предназначена конкретная модель, оставьте этот параметр **отключенным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) приведет к ошибкам и остановит работу дополнения.

Установите флажок **«Продвинутая маршрутизация моделей (под конкретные задачи)»**, чтобы получить детальный контроль. Это позволяет выбирать конкретные модели из выпадающего списка для разных задач:
- **Модель для OCR / Vision:** Выберите специализированную модель для анализа изображений.
- **Распознавание речи (STT):** Выберите конкретную модель для диктовки.
- **Синтез речи (TTS):** Выберите модель для генерации аудио.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Продвинутая настройка конечных точек (Провайдер Custom)
*Доступно только при выборе провайдера «Custom».*

> **⚠️ Предупреждение:** Этот раздел предназначен для ручной настройки API и разработан для **опытных пользователей**, запускающих локальные серверы или прокси. Неверные URL или имена моделей нарушат соединение. Если вы точно не знаете, что это за конечные точки, оставьте этот параметр **отключенным**.

Установите флажок **«Продвинутая конфигурация конечных точек»**, чтобы вручную ввести данные сервера. В отличие от нативных провайдеров, здесь необходимо **вводить** конкретные URL и имена моделей:
- **URL списка моделей:** Конечная точка для загрузки доступных моделей.
- **URL конечных точек для OCR/STT/TTS:** Полные URL для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Пользовательские модели:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.4 Общие настройки
- **Движок OCR:** Выберите между **Chrome (Быстрый)** для быстрых результатов и **Gemini (Структурированный)** для лучшего сохранения форматирования.
    - *Примечание:* Если вы выберете «Gemini (Структурированный)», но ваш провайдер установлен на OpenAI/Groq, дополнение интеллектуально перенаправит изображение на модель vision вашего активного провайдера.
- **Голос TTS:** Выберите предпочтительный стиль голоса. Этот список динамически обновляется в зависимости от вашего активного провайдера.
- **Креативность (Температура):** Управляет случайностью ответов AI. Более низкие значения лучше подходят для точного перевода/OCR.
- **URL прокси:** Настройте это, если AI-сервисы ограничены в вашем регионе (поддерживает локальные прокси, такие как `127.0.0.1` или bridge URL).

## 2. Командный слой и горячие клавиши

Во избежание конфликтов клавиш это дополнение использует **Командный слой**.
1. Нажмите **NVDA + Shift + V** (Главная клавиша), чтобы активировать слой (вы услышите звуковой сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих одиночных клавиш:

| Клавиша       | Функция                     | Описание                                                                 |
|---------------|-----------------------------|--------------------------------------------------------------------------|
| **T**         | Умный переводчик            | Переводит текст под курсором обзора или выделенный текст.                |
| **Shift + T** | Перевод из буфера обмена    | Переводит содержимое, находящееся в данный момент в буфере обмена.       |
| **R**         | Обработка текста            | Суммировать, исправить грамматику, объяснить или выполнить **Пользовательские запросы**. |
| **V**         | Распознавание объекта       | Описывает текущий объект обзора.                                         |
| **O**         | Анализ всего экрана         | Анализирует макет и содержимое всего экрана.                             |
| **Shift + V** | Анализ онлайн-видео         | Анализировать видео с **YouTube**, **Instagram**, **TikTok** или **Twitter (X)**. |
| **D**         | Чтение документов            | Расширенное средство чтения PDF и изображений с выбором диапазона страниц. |
| **F**         | OCR файла                   | Прямое распознавание текста из выбранного файла изображения, PDF или TIFF. |
| **A**         | Транскрибация аудио         | Транскрибировать файлы MP3, WAV или OGG в текст.                         |
| **C**         | Решение CAPTCHA             | Захватывает и решает CAPTCHA (поддерживаются правительственные порталы). |
| **S**         | Умная диктовка              | Преобразует речь в текст. Нажмите, чтобы начать запись, нажмите снова, чтобы остановить/ввести текст. |
| **L**         | Состояние работы            | Объявляет текущий прогресс (например, «Сканирование...», «Ожидание»).    |
| **U**         | Проверка обновлений         | Вручную проверить GitHub на наличие последней версии дополнения.         |
| **Пробел**     | Последний результат         | Показывает последний ответ AI в диалоговом окне для просмотра или уточнения. |
| **H**         | Помощь по командам          | Отображает список всех доступных ярлыков внутри командного слоя.         |

### 2.1 Горячие клавиши в средстве просмотра документов
- **Ctrl + Страница вниз:** Перейти к следующей странице.
- **Ctrl + Страница вверх:** Перейти к предыдущей странице.
- **Alt + A:** Открыть диалоговое окно чата, чтобы задавать вопросы о документе.
- **Alt + R:** Принудительное **повторное сканирование с AI** с использованием вашего активного провайдера.
- **Alt + G:** Создать и сохранить высококачественный аудиофайл (WAV/MP3). *Скрыто, если провайдер не поддерживает TTS.*
- **Alt + S / Ctrl + S:** Сохранить извлеченный текст в файл TXT или HTML.

## 3. Пользовательские запросы и переменные

Вы можете управлять запросами в **Настройки > Запросы > Управление запросами...**.

### Поддерживаемые переменные
- `[selection]`: Текущий выделенный текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Снимок экрана объекта обзора.
- `[screen_full]`: Снимок всего экрана.
- `[file_ocr]`: Выбрать файл изображения/PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, код, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех AI-функций требуется активное интернет-соединение. Многостраничные документы обрабатываются автоматически.
